랜딩 페이지란?

최근 IT 연합 동아리 Product 팀의 프론트엔드 개발자로 활동하면서, 다음 기수 리브랜딩에 따른 동아리 랜딩 페이지 리뉴얼 작업을 진행했습니다.
 
랜딩 페이지란 ‘전환’ 이라는 목표를 달성하기 위해 만들어진 웹 페이지입니다. 동아리 랜딩 페이지의 경우 ‘지원하기’ 라는 CTA(Call-To-Action)를 클릭하도록 유도하는 것이 가장 중요합니다. CTA 버튼을 페이지 전반에 반복해서 배치함으로써, 유저가 어느 영역을 보고 있더라도 CTA 버튼에 쉽게 접근할 수 있도록 하는 것이 좋습니다.

목표 설정

CTA 버튼에 쉽게 접근할 수 있는 것도 중요하지만, 유입된 유저를 이탈시키지 않는 것도 중요합니다. 기획자와 디자이너께서는 ‘빛’을 이용한 화려한 디자인을 통해 유저를 사로잡기 원하셨습니다. 따라서 해당 디자인을 코드로 잘 구현하는 것이 프론트엔드 개발자로서의 첫번째 목표였습니다.
 
두번째 목표는 모바일 친화적으로 제작하는 것이었습니다. 랜딩 페이지의 경우 모바일이 트래픽의 상당 부분을 차지한다고 합니다. 직접 GA(구글 애널리틱스)를 심어서 확인해봤을 때, 실제로도 모바일을 통해 접근하는 사용자의 비율이 더 높았습니다. 따라서 모바일에서도 어색하게 보이지 않도록 반응형을 신경쓰면서 제작하는 것이 중요합니다.
 
세번째 목표는 다음 기수 운영자가 쉽게 이해하고 재활용할 수 있는 코드 작성하기, 네번째 목표는 SEO 최적화하기 였습니다.
 
이번 글에서는 세번째 목표까지 달성하기 위해 고민했던 사항들과, 다음번 랜딩 페이지를 만들 때 재활용해볼만한 내용들을 정리해두려고 합니다.
 
이전 운영자분이 사용했던 코드 저장소를 이어서 사용했고, 사용한 기술 스택은 Gatsby와 SCSS입니다.
Gatsby는 SSG과 다양한 최적화 기법을 제공하고 있기 때문에 랜딩 페이지에 적합한 기술이라고 생각해 그대로 유지했습니다.
 

랜딩 페이지 재사용 코드 조각

다음 기수, 다다음 기수 리뉴얼 작업을 진행하더라도 재사용할 수 있는 코드 조각들을 만들고자 했습니다. 따라서 디자인 요구사항과 다양한 랜딩 사이트를 참고해보며 공통되는 요구사항을 분석해봤습니다.
 

#1 모바일과 데스크탑의 줄바꿈이 다르다면?

아래 예시처럼 데스크탑 모드에선 2줄이었던 헤더가 모바일에선 3줄로 보이길 원하시는 경우가 있었습니다.
 
첫번째로 고민했던 방식은 word-break 속성을 이용하는 것이었습니다. 이 속성을 이용하면 화면 너비에 따라 적절하게 줄바꿈을 해주지만, 정확히 원하는 부분에서 줄바꿈을 할 수 없다는 부족함이 있었습니다.
 
다음으로는 padding을 사용하여 모바일에서의 여백을 조정하는 방식을 고민했습니다. 그러나 마찬가지로 padding만으로는 정확히 원하는 지점에서 줄바꿈을 하기 어려웠습니다.
 
따라서 <br /> 태그를 적절히 사용해서 해결해보기로 결정했습니다.
 
설계는 아래와 같이 했습니다.
import clsx from 'clsx'

import * as css from './br.module.scss'

interface Props {
  mobile?: boolean
  desktop?: boolean
}

/**
 * 스크린 사이즈별 줄바꿈 컴포넌트
 *
 * @component
 * @param {boolean} mobile 모바일 사이즈에서 줄바꿈 여부 (*기본값: false)
 * @param {boolean} desktop 데스크탑 사이즈에서 줄바꿈 여부 (*기본값: false)
 *
 * @example
 * <Br mobile desktop /> // 모바일, 데스크탑 모두 줄 바꿈
 * <Br mobile /> // 모바일에서만 줄바꿈
 * <Br desktop /> // 데스크탑에서만 줄바꿈
 */
export const Br = ({ mobile = false, desktop = false }: Props) => {
  return <br className={clsx({ [css.br_visible_mobile]: mobile, [css.br_visible_desktop]: desktop })} />
}
mobile prop만 존재한다면 모바일에서만 줄바꿈을 진행하고, mobile, desktop props 둘 다 존재한다면 모바일과 데스크탑에서 모두 줄바꿈이 되도록 하는 방식입니다.
 
CSS는 아래와 같이 작성했습니다.
@use '@/styles/abstracts' as *;

.br_visible_mobile {
  display: none;

  @include mobile {
    display: block !important;
  }
}

.br_visible_desktop {
  display: none;

  @include desktop {
    display: block !important;
  }
}
처음엔 !important 키워드를 작성하지 않았는데, className="br_visible_mo br_visible_desktop" 일 때 모바일모드에서 display: none으로 작동하는 문제가 있었습니다. 뒤에 선언한 br_visible_desktop 클래스의 우선순위가 더 높기 때문에 발생했던 문제였습니다. 따라서 !important 키워드를 추가해줌으로써 문제를 해결했습니다.
 
실제 사용 코드한 코드는 아래와 같습니다.
const Title: ReactNode = (
  <>
    CMC 챌린저들은
    <Br mobile />
    각자가 <span>프로덕트의 주인공</span>이 되어
    <Br mobile desktop />
    새로운 길을 향해 도전합니다.
  </>
)
 

#2 스크롤이 해당 섹션에 위치했을 때 등장할 수 있도록 구현하기

랜딩 페이지를 살펴보면 화면에 포커스될 때 등장하는 효과의 애니메이션을 많이 보이곤 합니다. 이러한 방식의 UX는 유저가 해당 내용에 집중하도록 유도하는 효과를 줍니다.
 
어느 섹션이나 요소에든 내부 구현사항이 바뀌더라도 쉽게 활용할 수 있도록 선언적으로 작성하고자 했습니다. 
<Main className={css.main}>
  <포커스될때_등장하는_효과>
    <FeatureSection />
  </포커스될때_등장하는_효과>
  
  <포커스될때_등장하는_효과>
    <IdentitySection />
  </포커스될때_등장하는_효과>
	 
	 // ...

</Main>
/**
 * 스크롤이 해당 역영에 위치했을 때, 위로 올라오면서 등장하는 효과를 주는 컴포넌트
 *
 * before: opacity 0, translateY: 50px
 *  -> after: opacity: 1, translateY: 0px
 *
 * @component
 * @param {string} children 효과를 줄 자식 컴포넌트
 * @param {string} className 원하는 class가 있다면 추가할 수 있습니다.
 *
 * @example
 * <ScrollRevealContainer> // 효과를 줄 컴포넌트를 감싸주면 됩니다.
 *   <ExampleSection />
 * </ScrollRevealContainer>
 */
export const ScrollRevealContainer = ({ className = '', children }: Props) => {

	 // ...

  return <div className={clsx('animateBox', css.box)}>{children}</div>
}
이제 컴포넌트에 기능을 추가해보겠습니다.
 
JavaScript의 IntersectionObserver API를 사용하여 요소가 뷰포트에 들어오는 순간을 감지하고, CSS를 통해 애니메이션 효과를 적용할 예정입니다.
 
먼저 CSS 코드는 아래와 같습니다.
@keyframes reveal {
  0% {
    opacity: 0;
    transform: translate3d(0px, 50px, 0px);
  }
  100% {
    opacity: 1;
    transform: translate3d(0px, 0px, 0px);
  }
}

.box {
  opacity: 0;
}

.visible {
  animation: 0.5s ease-in-out 0s 1 normal forwards running reveal;
  opacity: 1;
}
transform 속성을 사용하여 위치 변경 애니메이션을 구현합니다. 기본 위치를 translateY(50px)로 설정하여 요소가 처음에는 아래쪽으로 내려가 있게 합니다. 요소가 뷰포트에 진입할 때 visible 클래스를 추가하여 translateY(0px)으로 돌아오는 애니메이션을 보여줍니다.
 
컴포넌트 코드는 다음과 같습니다.
export const ScrollRevealContainer = ({ className = '', children }: Props) => {
  useEffect(() => {
    const initObserver = () => {
      const options = {
        root: null,
        threshold: 0.2,
      }

      const observerCallback: IntersectionObserverCallback = (entries) => {
        entries.forEach((entry) => {
          if (entry.isIntersecting) {
            entry.target.classList.add(css.visible) // 요소가 뷰포트에 진입할 때 visible 클래스 추가
          }
        })
      }

      return new IntersectionObserver(observerCallback, options)
    }

    const myElements = document.querySelectorAll('.animateBox')
    const observer = initObserver()

    myElements.forEach((element) => {
      observer.observe(element)
    })
  }, [children])

  return <div className={clsx('animateBox', css.box, className)}>{children}</div>
}
위와 같이 구현함으로써 페이지 스크롤에 따라 요소가 자연스럽게 올라오는 애니메이션을 구현할 수 있었습니다.
 

#3 가로 스크롤 캐러셀

이전 기수의 작업물을 동적으로 보여주는 가로 스크롤 캐러셀을 구현하는 요구사항이 있었습니다. 디자인 요구사항에 따르면, 두 줄로 구성된 캐러셀에서 각 줄에 4.5개의 이미지가 보여야 했으며, 각 줄은 서로 반대 방향으로 슬라이드되어야 했습니다.
 
이 캐러셀을 구현하면서 가장 고민을 많이 한 부분은 어떻게 하면 추후 변경사항에 유연하게 대처할 수 있을지였습니다. 많은 고민 끝에 각 줄의 이미지 데이터, 이미지 너비, 그리고 슬라이드의 진행 방향이라는 세 가지 주요 변수를 컴포넌트의 props로 받기로 결정하였습니다.
 
SCSS를 사용하고 있기 때문에, HTML에서 직접적으로 변수를 받아 동적으로 스타일을 적용하는 것은 불가능했습니다. 이는 SCSS가 컴파일 시 처리되기 때문입니다. 따라서 이 한계를 극복하기 위해 두 가지 방법을 통해 비교했습니다.
 
  1. CSS 변수 사용: CSS 커스텀 속성을 통해 런타임에 스타일을 동적으로 변경할 수 있습니다. 이를 SCSS에서 선언하고 JavaScript를 사용하여 페이지 로드 후 또는 이벤트 발생 시 이 변수들을 조정할 수 있습니다.
      2. :root {
  --image-count: 19; // 기본 값
}

.carouselSlide {
  width: calc((12%) * var(--image-count)); // 이미지 하나의 너비 X 이미지 갯수
  animation: slide 50s linear infinite;
}
      document.documentElement.style.setProperty('--image-count', '15');
  1. 인라인 스타일 사용: React 같은 프레임워크를 사용할 경우, 컴포넌트에 인라인 스타일을 적용하여 더욱 동적으로 스타일을 관리할 수 있습니다. 이는 SCSS의 일부 기능을 사용할 수 없는 단점이 있지만, 컴포넌트의 상태나 props에 따라 스타일을 즉각적으로 변경할 수 있는 장점이 있습니다.
      2. const MyComponent = ({ imageCount }) => {
  return (
    <div
      style={{ width: `calc((12% + 8px) * ${imageCount})` }}
    >
      {/* 아이템들 */}
    </div>
  );
};
 
SCSS에서 직접 HTML이나 JavaScript의 변수를 받아 처리할 수는 없으나, CSS 커스텀 속성이나 인라인 스타일을 통해 유사한 결과를 달성할 수 있습니다. 이들 방법은 유연한 웹 개발 환경을 제공하며, 다양한 디자인 시스템과의 상호작용을 가능하게 합니다.
 
다음은 로고 이미지 배열을 사용하여 캐러셀을 구성하는 CarouselSlider 컴포넌트의 예시입니다. 이 컴포넌트는 로고들을 캐러셀 형식으로 표시하며, 필요에 따라 슬라이드의 방향을 반대로 할 수 있습니다. 끊김 없는 루프 효과를 위해 시야에 보이는 개수만큼의 항목을 끝에 추가하여, 사용자 경험을 개선하였습니다
// CarouselSlider.tsx

import * as css from './carousel-slider.module.scss'
// ...

interface Props {
  logos: IGatsbyImageData[]
  width?: number | string
  reverse?: boolean
}

/**
 * 로고들을 캐러셀 형식으로 표시하는 CarouselSlider 컴포넌트입니다. 자동 무한루프 방식으로 동작하며 필요에 따라 방향을 반대로 할 수 있습니다.
 *
 * @component
 * @param {IGatsbyImageData[]} logos 로고 이미지 배열
 * @param {number | string} width 캐러샐의 각 아이템의 너비 (기본값: 10%)
 * @param {boolean} reverse true인 경우 캐러샐은 반대 방향으로 슬라이드됩니다. (기본값: false)
 *
 * @example
 * <Carousel logos={imageArr} /> // 순방향
 * <Carousel logos={imageArr} reverse /> // 역방향
 */
export const CarouselSlider = ({ logos, width = '10%', reverse = false }: Props) => {
  const extendSlideItems = [...logos, ...logos.slice(0, 4)] // 끊김 없는 루프 효과를 위해 시야에 보이는 개수인 4개의 항목을 끝에 추가합니다.
  const carouselRef = useRef<HTMLDivElement>(null)

  useEffect(() => {
    if (carouselRef.current) {
      // 로고 데이터의 길이와 지정된 너비에 따라 동적으로 캐러셀 스타일을 조정합니다.
      carouselRef.current.style.setProperty('--slide-item-length', `${logos.length}`)
      carouselRef.current.style.setProperty('--slide-item-width', `${width}`)
    }
  }, [logos.length, width])

  return (
    <div ref={carouselRef} className={clsx(css.carousel_container, { [css.carousel_container__reverse]: reverse })}>
      {extendSlideItems.map((logo, idx) => (
        <GatsbyImage
          // eslint-disable-next-line react/no-array-index-key
          key={idx}
          className={css.carousel_item}
          image={logo}
          alt="프로덕트 로고 이미지"
          objectPosition="50% top"
          loading="eager"
        />
      ))}
    </div>
  )
}
@use '@/styles/abstracts' as *;
@use '@/styles/utils' as *;

$gap: 8px;

@keyframes slide {
  0% {
    transform: translateX(0);
  }
  100% {
    // 슬라이드 아이템의 너비와 간격을 고려하여 전체 슬라이드를 왼쪽으로 이동시키는 계산을 수행합니다.
    transform: translateX(calc((var(--slide-item-width) + $gap) * -1 * var(--slide-item-length)));
  }
}

.carousel_container {
  display: flex;
  gap: $gap;
  // 컨테이너의 너비를 각 슬라이드 아이템의 너비와 총 아이템 수를 고려하여 동적으로 계산합니다.
  width: calc((var(--slide-item-width) + $gap) * var(--slide-item-length));
  animation: 60s linear infinite running slide;

  .carousel_item {
    flex: 0 0 auto;
    width: var(--slide-item-width);
    aspect-ratio: 416 / 280;
    border-radius: 8px;
  }
}

.carousel_container__reverse {
  animation: 60s linear infinite reverse slide;
}
